# Methodology: 写作方法论

## 完整写作流程

### 0. 理解任务
- 话题/选题、角度、目标字数由 operator 指定
- 短篇(1000-2000字) / 中篇(2000-5000字) / 长文(5000字+)，默认中篇

### 1. 调研（仅首次，后续节制加载参考文件）
- 查证关键事实、时间线、主要参考来源
- 注意区分：已确认 / 据报道 / 官方声称 / 分析推断
- 调研完成后不允许反复 reload 参考文件

### 2. 计划（一页内完成）
- 拟定核心判断（一句话）
- 拟定段落结构（3-6段）
- 确认与已有文章的角度区分

### 3. 写作（连续输出，先写完再改）
- 从具体触发物开始，不写"在当今时代"式开头
- 按段落弧线推进：触发 → 判断 → 反框架 → 语境 → 回人 → 余味
- 写完整篇初稿前不中断去审校中间段落

### 4. 自我审校（最后一步）
- 按 `quality-review.md` 逐条检查
- 改完再 commit，不边写边改

## 关键原则

- **先写完整初稿，再逐条审校** — 不要在写作中途停下来检查格式
- **参考文件只读一次** — 调研完成后，写作阶段不重读所有参考文件
- **一次只改一件事** — 审校时逐条通过，不要同时改多处
