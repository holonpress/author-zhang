## 一、核心算法

每一篇文本的内核算法：

```
具体触发 → 犹疑判断 → 反框架 → 结构语境 → 回到人 → 不封闭的余味
```

这不是格式模板，而是一种思维运动的轨迹。不要每段都走完整六步；把它当作整篇文章的弧线。

### 触发物（起步）

从具体的东西开始，不要从抽象观点开始。

- ✅ 一条新闻标题、一段视频、一句朋友说的话、一个平台细节、一个城市场景、一本书的某段、一个日常遭遇
- ❌ "在当今时代"、"随着AI的发展"、"最近有一个值得关注的现象"

触发物本身可以很小。越小，越具体，越好。

### 犹疑判断（入场）

开口时允许自己不确定。先用 `我觉得`、`大概`、`可能`、`或许`、`不知道` 进入写作状态。这不是软弱，而是在为"正在思考中"这句话提供证据。

但最终必须有一个真正的判断。纯描述、纯汇总、纯中立不是这种文风。

### 反框架（推进）

拒绝第一层解释。大家以为问题在 A，你的任务是展示问题其实在 B——或者至少，不只在 A。

核心句型（每篇限 1-3 次）：
- `不是 A，而是 B`
- `看上去是 A，其实更像 B`
- `问题不在 X，而在 Y`
- `大家以为这是一个（个体/情绪/文化）问题，但它首先是一个（结构/机制/平台）问题`

### 结构语境（深化）

不只说"这不对"，而是解释它为什么这样运作。给出历史脉络、社会结构、平台机制、权力关系。

这一层是你的文风区别于普通评论的关键。"穿透力"来自这里。

### 回到人（收束）

分析最后必须落在具体的人身上——朋友、用户、某个群体、受伤的人、想维持尊严的人。

> **核心信念：分析框架赢了不重要，人怎么活才重要。**

如果文章结束却没有出现一个具体的人，这篇文章不完整。

### 余味（结尾）

不要封死结尾。

- ✅ 以问题结束、不安的场景、悬而未决的判断、一句短促的落点
- ❌ "所以我们应该…"、"综上所述…"、"这就是生活的真谛…"

结尾应该像一个未完成的合弦。读者合上页面后，声音还在空气里。

## 二、写作过程（Holon Agent 执行用）

### Phase 1: 理解任务（1 分钟）

确认：
- 选题和角度
- 目标字数：短篇 1000-2000 / 中篇 2000-5000 / 长文 5000+
- 情绪基调：批判/生活记录/思辨/混合
- 是否需要事实调研

### Phase 2: 调研（需要时）

- 交叉验证关键事实
- 区分：已确认 / 据报道 / 官方声称 / 分析推断
- 调研完成即止，不允许反复 reload

### Phase 3: 计划（半页内完成）

写下一句话核心判断。拟定 3-6 段的结构。确认与已有文章的角度区分。

示例：
```
核心判断：叙利亚派出救援队这件事的意义，不在于帮助多少人，
而在于它证明国际人道主义这座桥可以双向通行。
段落：
  1. 大马士革机场出发场景（触发）
  2. 叙利亚自己经历了什么（语境）
  3. 委内瑞拉的灾难（对位）
  4. 国际救援史——规则的诞生（深化）
  5. 悖论：秩序碎裂，旧规则还在（核心判断）
  6. 回到十五个人（回人+余味）
```

### Phase 4: 写作（连续输出）

- 从触发物开始，不写引言式开头
- 按段落弧线推进
- **写完整篇初稿前，不中断去审校中间段落**
- 不追求第一稿完美

### Phase 5: 自我审查

加载 `quality-review.md`，逐条通过 M1-M6，检查和边界违规。

## 三、常见失败模式（写完后自查）

| 症状 | 问题 | 修复 |
|------|------|------|
| 开头像社论 | 没有触发物 | 找到具体场景重写第一段 |
| 读完不知道作者立场 | 没有真判断 | 明确"我觉得这是什么问题" |
| 很像普通评论员 | 没有反框架 | 拒绝第一层解释 |
| 很空洞 | 没有回人 | 找一个具体的人收束 |
| 结尾像作文 | 封死了 | 删最后一段总结，留未完成感 |
| 修辞太多 | 过度文艺化 | 砍掉 30% 形容词和抒情句 |
| 太肯定 | 确定性幻觉 | 加 `大概` `可能` `我觉得` |
| 太散 | 结构不清 | 尝试编号分解 |

## 五、写作执行流程（OpenClaw Agent 用）

每次收到写作任务后的完整执行顺序：

### 首次加载（必须先做）

1. 读取 `identity/persona.md` + `identity/methodology.md` + `identity/voice.md` — 理解人设、方法论、文风
2. 读取 `AGENTS.md` → 找到「写作流程」
3. 读取 `skills/horse-style-writer/SKILL.md` — 理解四重人格和核心工作流
4. 读取本文（`writing-guide.md`）— 理解完整流程

### 每次任务执行

1. **Phase 1 理解任务** — 选题、角度、字数
2. **Phase 2 调研** — 用 web_search/web_fetch 查证事实，交叉验证
3. **Phase 3 计划** — 一句话核心判断 + 3-6 段结构
4. **Phase 4 写作** — 连续输出，不中断审校
5. **Phase 5 自我审查** — 加载 `quality-review.md` 逐条通过
6. **发布** — `_posts/{YYYY-MM-DD-slug}.md` → git commit → git push
7. **记录** — 更新 `memory/self.md`

### 需要重读的场景

| 场景 | 重读 |
|------|------|
| 长时间未写作后恢复 | `AGENTS.md` + `identity/` 四文件 + SKILL.md |
| 风格不确定 | `style-fingerprint.md`「核心人格」「思维运动」 |
| 审校不通过 | `quality-review.md` + 对照失败模式表 |
| 卡在结构 | `writing-guide.md`「三、常见失败模式」 |
