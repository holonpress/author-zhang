# Writing Guide（写作参考）

**本文件不是流程定义。写作流程的唯一权威在 `AGENTS.md` →「写作循环」8 阶段。**
本文件提供各阶段的详细技术参考和手法说明。

---

## 一、Phase 1 热身参考

热身时读完 identity/ 四文件后，写一段热身笔记。格式参考：

```markdown
## 热身笔记 — {标题}

今天侧重角色：结构观察者 / 生活记录者 / 混合
核心判断（一句话）：…
触发物：从什么具体场景/新闻/对话开始
预期反框架：大家以为的问题是X，但其实是Y
```

**不要跳过热身。热身笔记是 Phase 3 Plan 的前置条件。**

---

## 二、Phase 3 Plan 段类型参考

每段必须标一个类型。类型不是越多越好，是越准越好。

| 类型 | 做什么 | 不要做什么 |
|------|--------|-----------|
| **trigger** | 写具体场景：时间/地点/人/动作 | 不要开头讲道理 |
| **judgment** | 给出犹疑但明确的判断 | 不要纯描述/中立汇总 |
| **reverse_frame** | 拒绝表层解释，给出替代 | 不要每段都用 `不是A而是B`（整篇 ≤ 3 次） |
| **context** | 放入历史/结构/机制 | 不要只停留在当下 |
| **people** | 回到具体的人/群体 | 不要只有分析没有温度 |
| **aftertaste** | 留开放式结尾 | 不要"综上所述" |

**Plan 示例：**

```
核心判断：叙利亚派出救援队这件事的意义，不在于帮助多少人，
而在于它证明国际人道主义这座桥可以双向通行。

段落结构：
  1. trigger  — 大马士革机场出发场景
  2. context  — 叙利亚自己经历了什么（内战/制裁/崩溃）
  3. context  — 委内瑞拉的双震灾难（对位）
  4. reverse_frame — 不是"南南合作"，而是"被救者伸出援手"
  5. people   — 回到十五个救援队员
  6. aftertaste — 大西洋上空的那根线
```

## 三、Phase 4 写作手法参考

### 常用句子结构（节制使用）

- `不是 A，而是 B` — 整篇 ≤ 3 次
- `我觉得` / `大概` / `可能` / `或许` — 2-5 次为佳
- `问题不在 X，而在 Y` — 替换 `不是A而是B` 的好选择
- `但` / `只是` / `不过` — 段落内深化转折
- 编号分解 — 复杂问题用 1. 2. 3.；不要为了有条理而编
- 括号补充 — 每段 ≤ 2 处
- 短句结尾 — 长段后用短句砸一下

### 绝对不要

- ❌ "在当今时代"、"随着…的发展"式开头
- ❌ "让我们一起"、"相信未来"式励志结尾
- ❌ "这才是真相"、"事实很清楚"式绝对断言
- ❌ 编造引语、数据、私人经历

## 四、Phase 5 审查参考

审查分三个维度：

| 维度 | 检查项 | 通过标准 |
|------|--------|---------|
| 思维运动 (M1-M6) | 见 quality-review.md | ≥ 4 Yes |
| 修辞节制 (R1-R4) | 频率检查 | 不超标 |
| 边界合规 (B1-B4) | 违规检查 | 零违规 |

此外增加事实检查：
- 每个关键数据有来源？
- 区分了"据报道"和"已确认"？
- 来源可追溯？（文末有链接列表）

## 五、常见失败模式

| 症状 | 问题 | 修复 |
|------|------|------|
| 开头像社论 | 没有触发物 | 找到具体场景重写第一段 |
| 读完不知道立场 | 没有真判断 | 明确"我觉得这是什么问题" |
| 很像普通评论员 | 没有反框架 | 拒绝第一层解释 |
| 很空洞 | 没有回人 | 找一个具体的人收束 |
| 结尾像作文 | 封死了 | 删最后一段总结，留未完成感 |
| 修辞太多 | 过度文艺化 | 砍掉 30% 形容词和抒情句 |
| 太肯定 | 确定性幻觉 | 加 `大概` `可能` `我觉得` |
| 太散 | 结构不清 | 回 Phase 3 重写 Plan |
| 数据不可查 | 来源缺失 | 加超链接来源列表 |
