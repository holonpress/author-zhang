# 文风质量审查表

逐条检查你的文章。每条用 Yes / No / Partial 标记。

---

## 一、思维运动检查（核心）

### M1. 触发物
文章是否从一件具体的事情开始？不是从抽象观点开始的。

- ✅ 好的：一条新闻标题、一句朋友说的话、一个咖啡馆的细节、一个平台上的遭遇
- ❌ 不好的：直接开始讲道理、引用统计数据开头
- 🔧 不通过 → 重写第一段，找个具体的触发物。参考 `skills/horse-style-writer/references/examples/短判断.md` 和 `生活记录.md`

### M2. 有一个真正的判断
文章有一个自己的判断——即使是带着 `我觉得` `大概` `可能` 的。

- ✅ 好的：判断很明确，即使带着犹疑
- ❌ 不好的：纯描述、中立汇总、态度模糊
- 🔧 不通过 → 问自己：这篇文章里，我到底在说"我觉得这是什么问题"？

### M3. 反框架
拒绝了第一层解释。推动了"大家以为的问题是X，但其实是Y"。

- 检查：有没有 `不是A，而是B` / `看上去是A，其实更像B` / `问题不在X，而在Y`
- 🔧 不通过 → 对照 `skills/horse-style-writer/references/style-fingerprint.md`「论证结构」节，看示例中如何找到反向框架

### M4. 语境化
不只是说"这不对"，而是解释了它为什么这样运行。

- ✅ 好的：有历史脉络、结构分析、机制拆解
- ❌ 不好的：只停留在当下事件的表层
- 🔧 不通过 → 问自己：这个问题放到更长的历史里看是什么样子？背后有什么机制在运转？参考 `skills/horse-style-writer/references/examples/结构化推理.md`

### M5. 回到人
分析最后落到了具体的人身上。

- 检查：文章中是否有具体的"人"出现——用户、朋友、某个群体、受伤的人、普通人
- 🔧 不通过 → 问自己：这个分析如果不落地，对谁都没有意义。找个具体的人或处境来收束。参考 `skills/horse-style-writer/references/style-fingerprint.md`「情绪结构」

### M6. 结尾不封死
结尾是开放的、有余味的，不是金句式总结。

- ✅ 好的：以问题结束、不安的场景、悬而未决的判断
- ❌ 不好的："所以我们应该…"、"综上所述…"、"这就是生活的真谛…"
- 🔧 不通过 → 删掉最后一段总结性的话，换成更有余味的东西。参考 `skills/horse-style-writer/references/examples/自我拆台与余味.md`

## 二、修辞检查

### R1. `不是A而是B` 使用频率
- 1-3 次：✅
- 4 次以上 → 替换其中一两个
- 0 次 → 检查是否有一个地方可以用它来推动分析

### R2. `我觉得` / `大概` / `可能` 犹疑词
- 2-5 次：✅
- 0 次 → 可能写得太肯定了
- 8 次以上 → 考虑加强判断力度

### R3. 编号拆解
- 如果是真正的结构化需要：✅
- 如果只是为了"看起来有条理" → 去掉或合并

### R4. 括号注释
- 每段 ≤ 2 处：✅
- 一页 > 3 处 → 削减

---

## 三、边界检查

### B1. 广告/自媒体语气
- ❌ "必看" "震惊" "建议收藏" "作为一个XX，我…"
- 🔧 有 → 改写成平实的语气

### B2. 过度文艺化
- ❌ "风很轻，云很淡"式抒情、堆砌修辞
- 🔧 有 → 删掉多余的形容词和抒情句

### B3. 确定性幻觉
- ❌ "这才是真相" "事实很清楚" "所有人都知道"
- 🔧 有 → 加限定词：`可能有` `我倾向于认为` `至少现阶段看来`

### B4. 编造
- ❌ 虚构的引语、数据、个人经历、朋友的话、来源不明的"据专家称"
- 🔧 有 → 删除。诚实标注"我记得"或"印象中"

---

## 四、总评

| 项目 | 评分 | 不通过时的修复 |
|------|------|---------------|
| 触发物 (M1) | | 重写第一段 |
| 真判断 (M2) | | 明确立场 |
| 反框架 (M3) | | 对照指纹「论证结构」 |
| 语境化 (M4) | | 加历史/结构/机制层 |
| 回到人 (M5) | | 找个具体人收束 |
| 余味结尾 (M6) | | 删总结，留压力 |
| 修辞节制 (R1-R4) | | 对照频率限制调整 |
| 边界合规 (B1-B4) | | 删除越界内容 |

**通过：M1-M6 ≥ 4 Yes + 无边界违规。否则修改后重审。**
