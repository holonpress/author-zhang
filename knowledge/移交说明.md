# Horse 写作研究移交包

## 这是什么

这是一个供其他 AI Tool（AI 工具）继续使用的精简移交包。内容来自对 Horse 本人多年 Twitter/X 公开写作记录的整理与研究。

本包不包含完整 Twitter Archive（推特原始归档），只保留经过筛选、适合研究写作风格的 Highlights（精选内容）、写作人格与文风指纹报告，以及由研究结论形成的 Writing Skill（写作技能）。

## 文件结构

- `Highlights/`：628 条精选 Markdown 文档，按 2007–2026 年分目录保存。
- `Research/写作人格与文风指纹研究.md`：对写作人格、句法节奏、修辞方式、情绪结构与风格边界的完整研究报告。
- `Skill/horse-style-writer/`：可供 Codex 或其他支持 Markdown Prompt（提示词）/Skill（技能）机制的 AI 工具读取的写作技能。

## 推荐读取顺序

1. 先读 `Research/写作人格与文风指纹研究.md`，理解研究结论与适用边界。
2. 再读 `Skill/horse-style-writer/SKILL.md` 和 `references/style-fingerprint.md`，了解实际写作流程。
3. 需要核对风格依据、寻找真实语境或重新校准时，再读取 `Highlights/` 中的 Markdown 文档。

## 使用原则

- 将 Style Fingerprint（文风指纹）视为思考结构与表达习惯，不要只机械复用口头禅。
- 不要根据历史帖子虚构新的个人经历、关系、地点或观点。
- 写新内容时，以使用者当前提供的事实和立场为准。
- 如果其他工具不支持 Skill 目录，可把 `SKILL.md` 与 `references/style-fingerprint.md` 作为 System Prompt（系统提示）或长期上下文使用。

## 明确未包含

- 全量帖子、转发与媒体附件
- CSV、JSON 等中间数据
- 构建 Obsidian Vault（知识库）的程序与临时分析文件
